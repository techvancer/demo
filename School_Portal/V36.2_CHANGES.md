## 📊 Data Flow

```
Teacher clicks "Upload Marks" (Blue button)
    ↓
UploadMarks page loads
  - Fetches exam details
  - Loads all students
  - Counts questions for exam/employee/subject/section
    ↓
Teacher uploads marks (CSV or manual entry)
  - Downloads template or types directly
  - System validates (0 or 1 only)
  - Shows error if invalid
    ↓
Teacher clicks "Submit & Start Workflow"
  - Shows confirmation modal
    ↓
Confirmation clicked
  - sendMarksToWorkflow() called
  - Data sent to n8n webhook
  - Webhook stores to studentanswers_tbl
  - Exam status updated to "completed"
    ↓
Success message shown
  - Redirects to exams page
  - Teachers sees "Completed" badge
```

## 📋 CSV Template Format

Teachers can download a template that looks like:

```
studentid,studentname,q1,q2,q3,q4,q5
1,Ali Mohammed,,,,,
2,Omar Ahmad,,,,,
3,Yousef Khalid,,,,,
4,Hassan Nasser,,,,,
5,aqeel,,,,,
```

Teachers fill in:
```
studentid,studentname,q1,q2,q3,q4,q5
1,Ali Mohammed,1,0,1,1,0
2,Omar Ahmad,0,1,1,0,1
3,Yousef Khalid,1,1,0,1,1
4,Hassan Nasser,1,0,1,1,1
5,aqeel,0,1,1,0,0
```

Where:
- `1` = Correct answer
- `0` = Incorrect answer
- `` (empty) = Not answered

## 🐛 Troubleshooting

### Issue: "Papa Parse not found"
**Solution:** Run `npm install`

### Issue: "Upload Marks button not showing"
**Solution:** Make sure exam status is "new" (not "completed")

### Issue: CSV upload fails
**Solution:** Check CSV format matches template (studentid, studentname, q1, q2...)

### Issue: Validation error for valid marks
**Solution:** Make sure marks are exactly "0" or "1" (not "0.0" or "yes")

### Issue: n8n webhook doesn't receive data
**Solution:** Check webhook URL in `src/lib/n8nWebhook.js`

### Issue: Page shows blank after upload
**Solution:** Check browser console (F12) for errors
